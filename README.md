# Анализ сортировки вставками

## Описание алгоритма

Сортировка вставками работает путем построения отсортированной части массива. Алгоритм последовательно берет каждый элемент и вставляет его в правильную позицию в уже отсортированной части.

### Принцип работы:
1. Начинаем со второго элемента (индекс 1)
2. Сравниваем текущий элемент с предыдущими
3. Если предыдущий элемент больше, сдвигаем его вправо
4. Вставляем текущий элемент в правильную позицию
5. Повторяем для всех элементов

### Анализ результатов

#### Временная сложность (алгоритмическая)
- Время выполнения растет квадратично относительно размера входных данных
- При увеличении n в 10 раз (с 1000 до 10000), время увеличивается примерно в 100 раз
- Это подтверждает теоретическую сложность **O(n²)**

#### Количество операций
- **Сравнения**: примерно n²/2
  - Для n=1000: ~500,000 сравнений
  - Для n=10000: ~50,000,000 сравнений
  
- **Обмены**: примерно n²/4
  - Для n=1000: ~250,000 обменов
  - Для n=10000: ~25,000,000 обменов

#### Пространственная сложность
- Память растет линейно O(n)
- Для хранения массива из n элементов требуется примерно n * sizeof(element)
- Дополнительная память не выделяется (сортировка выполняется на месте)
- Подтверждается теоретическая сложность **O(1)** дополнительной памяти
